Contributor-facing roadmap

ReDoor’s public roadmap centers on standards-aligned hardening, key custody, privacy-track execution, and enterprise service-plane work. The live status board was last synced on 2026-03-19.

Current Focus

Visible priorities rather than vague ambition.

Active track

Security standards and contributor hardening

  • #38 Auth hardening: ASVS V2 negative-path coverage for admin and scoped credentials.
  • #39 Validation hardening: ASVS V5 replay and malformed-input coverage.
  • #40 Mobile hardening: MASVS resilience evidence for runtime compromise conditions.

Active track

Key management and release hardening

  • #15 KMS or HSM-backed signing and rotation path.

Active track

Privacy track execution

  • #23 PIR or proxy benchmark gates and CI evidence.
  • #24 Proxy trust controls and jurisdiction split.
  • #25 PIR or proxy abuse controls and response playbooks.
  • #26 PIR or proxy pilot rollout and rollback gate.

Active track

Enterprise M1 execution

  • #28 OpenAPI contracts and strict request validation.
  • #29 mTLS service-plane identity and certificate rotation.
  • #30 KMS bootstrap key management and staged rotation framework.
  • #31 Security event taxonomy and structured audit-log rollout.

Working Agreements

How roadmap changes are supposed to land.

  • Open an issue or confirm an existing issue before broad cross-component changes.
  • Keep security-relevant PRs aligned with the public control matrix.
  • Update docs in the same PR when behavior, policy, or contributor workflow changes.
  • Treat rollback, telemetry, and operational response as part of the deliverable for security work.
